markdown语法插入代码

Copy from the web page about the markdown syntex. Let’s see how html would render these codes.

代码

如果要标记一小段行内代码，你可以用反引号把它包起来（`），例如：

Use the printf() function.
会产生：

Use the printf() function.

如果要在代码区段内插入反引号，你可以用多个反引号来开启和结束代码区段：

There is a literal backtick (`) here.
这段语法会产生：

There is a literal backtick (`) here.

代码区段的起始和结束端都可以放入一个空白，起始端后面一个，结束端前面一个，这样你就可以在区段的一开始就插入反引号：

A single backtick in a code span: `

A backtick-delimited string in a code span: `foo`
会产生：

A single backtick in a code span: `

A backtick-delimited string in a code span: foo


在代码区段内，& 和尖括号都会被自动地转成 HTML 实体，这使得插入 HTML 原始码变得很容易，Markdown 会把下面这段：


Please don’t use any <blink> tags.
转为：

Please don’t use any <blink> tags.


你也可以这样写：


&#8212; is the decimal-encoded equivalent of &mdash;.
以产生：

&#8212; is the decimal-encoded equivalent of &mdash;.

Ansys安装遇到的问题

1. 不能生成证书文件

按照教程进行安装，本该在运行证书生成器后，在当前文件夹内生成证书文件，但双击证书生成器后遇到错误”…cannot write into …”
提示”不能写入”，并没有生成证书文件

机智的强哥很快发现错误是由于当前文件夹是通过虚拟光驱加载，只读不能写入
将证书生成器移动到桌面再次运行，成功生成文件。

2. Invalid License

先按照youtube视频中示例的一个方法：
将License Manager先关闭”stop…”，再开启”start…”
完成后，重新打开ansys workbench，依旧提示错误

遂卸载重装。

现有 ansys16.0 和 ansys14.5 两种软件资源，破解方法不同。ansys14.5 安装后发现以上错误，所以现决定安装16.0版本。

运行ansys16.0安装程序setup，自动退出。于是无奈决定再次安装14.5版本，并不按照原教程中安装License Manager而改用16.0版本的破解方法

一个小白想当然的移花接木———v14.5的软件，v16.0的破解

安装完毕，按教程将”Sharefile”和”v160”两个文件夹用资源中的文件进行替换
此时发现所安装的 v14.5的软件中没有文件夹v160只有v145。

重命名，替换。
将资源中的v160重命名为v145

替换完毕运行workbench。
竟然运行成功无任何错误提示……

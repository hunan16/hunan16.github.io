hexo部署到github遭遇的坑(ssh)

Error

用hexo搭建个人博客，按照某不靠谱教程，进行至部署到github一步时终端提示错误：

鉴于在尝试各种解决办法的过程中，错误提示还有过各种变化，但总之就是不停报错…仅列举以上两个比较能反映问题的提示

Solution：SSH

又是机智的强哥，在我看了某解决方案打算安装较低版本git时及时制止了悲剧发生(其实已经手快安完了… 无奈又换回了最新版本)

强哥说：你的git没有hexo的密钥啊，怎么能连接成功呢
(大意如此)

于是我们开始配置ssh

博客(.io)文件夹下，进入git bash

(原谅只会用命令行的本小白，都不知git bash为何物… 只觉得，哇这个和命令行好像的什么鬼… git也有命令行的吗？？

生成的密钥指纹保存在/c/Users/Q/.ssh/id_rsa.pub

复制 the key fingerprint
进入自己的github
–>settings–>SSH and GPG keys–>New SSH key–>Add SSH key

回到bash

提示 successfully authenticated

没提示
完成配置，此时注意hexo配置文件中的仓库名用ssh形式表示而非http

部署
